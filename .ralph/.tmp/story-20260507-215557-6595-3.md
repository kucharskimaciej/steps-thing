Status: in_progress

Description:
Phase context:

# Feed Search And Practice Context

The feed is the primary app experience. It loads the authenticated user's steps, applies search filters and sort order, renders a virtualized video feed, records views, and records practice.

Search behavior is fully specified in `REWRITE_SPEC.md` section 10. Practice duplicate prevention is based on local day and optional collection/session ID.

Goal:

Implement the main `/feed` page with virtualized video cards and step actions.

Implementation:

1. Create `/feed` page.
2. Query authenticated user's steps.
3. Apply current search/filter/sort.
4. Render a virtualized list of feed cards.
5. Scale primary video height to feed width using stored video dimensions.
6. Cap video height around `60vh`.
7. Mount/play videos only when visible using intersection detection or virtualization visibility.
8. Feed card displays:
   - Primary video.
   - Step name.
   - Tags.
   - Variation button with count if variations exist.
   - Practice action.
   - Options menu.
9. Variation button opens overlay feed of related variations.
10. Options menu includes:
    - Copy link.
    - Show variations when available.
    - Edit.
    - Cancel/close.
11. Edit opens inline edit modal.

Tracer/phase plan:

1. Tracer: `/feed` lists owned steps with primary video card.
2. Add search/filter/sort integration.
3. Add virtualization/intersection playback and stable video sizing.
4. Add tags, variation modal, practice action, options menu.
5. Wire copy link and inline edit entry.
